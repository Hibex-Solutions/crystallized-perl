## Relevância
Metodologia de facto para desenvolvimento de serviços cloud-native: doze práticas que
cobrem codebase, dependências, configuração, serviços de apoio, builds, processos,
port binding, concorrência, descartabilidade, paridade de ambientes, logs e processos
administrativos. É a referência arquitetural central deste projeto — toda decisão de
stack deve ser avaliada contra estes fatores. Mencionada explicitamente no brief de
identidade visual como fundamento conceitual do mascote.
